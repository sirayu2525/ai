# Code Review Pipeline

## 使うとき

- 既存差分や既存ファイルのレビューを依頼したいとき
- 問題点、回帰リスク、テスト不足を見つけたいとき

## 使う基本 skills

- `research`
- `review`

## 進め方

- 以下は代表フローであり、レビュー中に前提確認が必要なら `research` に戻るなど、基本 skills を自由に行き来してよい。
- 特に、より良い実装方法、ライブラリ選定、ベストプラクティス確認が必要なら `research` で補強してから `review` に戻る。

1. `research` でレビュー対象と前提を把握する。
2. `review` で findings を優先して整理する。
3. 指摘がない場合も、そのことと残るリスクを返す。

## 依頼例

- 「この PR をレビューして」
- 「この差分の危ない点を見て」
