# Browser Analysis Pipeline

## 使うとき

- Web サイトや Web アプリの不具合をブラウザ上で再現して調べたいとき
- console、network、DOM、performance、認証状態から原因を掘りたいとき

## 使う基本 skills

- `research`
- `browser-debugging`

## 進め方

- 以下は代表フローであり、調査中に追加の前提確認が必要なら `research` に戻るなど、基本 skills を自由に行き来してよい。

1. `research` で対象 URL、症状、再現条件を整理する。
2. `browser-debugging` で一次情報を取り、原因を絞る。
3. 根拠、原因、未確定事項を分けて返す。

## 依頼例

- 「この画面でボタンが効かない原因を見て」
- 「ログイン後の挙動が変なので調べて」
