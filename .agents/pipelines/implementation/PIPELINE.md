# Implementation Pipeline

## 使うとき

- コード、設定、ドキュメントなどを変更したいとき
- 実装だけでなく、最低限の確認までまとめて進めたいとき

## 使う基本 skills

- `research`
- `creation`
- `verification`

## 進め方

- 以下は代表フローであり、途中で前提不足や新しい論点が出たら `research` に戻るなど、基本 skills を自由に行き来してよい。
- 特に、仕様の曖昧さ、既存挙動の確認、より良い実装方法やベストプラクティス確認が必要なら `research` に戻る。

1. `research` で対象、制約、既存文脈を把握する。
2. `creation` で必要な変更を作る。
3. `verification` で妥当な確認を行う。
4. 変更内容、実施した確認、未実施の確認を返す。

## 依頼例

- 「この不具合を直して」
- 「このページを作って」
- 「この設定を整理して」
