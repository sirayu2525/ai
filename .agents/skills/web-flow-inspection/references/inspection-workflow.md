# Inspection Workflow

## 着手前

- 対象 URL、主要な画面、見たいユーザーフローを整理する。
- ログイン要否、テスト用データ、モバイル確認の要否を確認する。
- Playwright MCP が使えるかを確認する。

## 何を見るか

### 情報設計

- 何が重要な画面か初見で分かるか。
- CTA が見つけやすいか。
- 情報量や順番に違和感がないか。

### 操作フロー

- 会員登録、検索、購入、送信などの主要フローで詰まらないか。
- 手戻りや迷いが発生しないか。
- フォームの入力補助やエラー表示が分かりやすいか。

### 状態変化

- loading、empty、error、success の状態が分かるか。
- 処理中に誤解を生む表示が出ないか。
- レスポンシブで崩れたり押しにくくなったりしないか。

## まとめ方

- 問題が起きた操作手順を具体的に書く。
- 体験上の違和感と、どの画面で起きたかを対応づける。
- 原因が技術的に曖昧なら `browser-debugging` に引き継ぐ。
