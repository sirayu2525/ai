# Creation

## 概要

成果物を作る、または更新するための基本 Skill です。コード実装だけでなく、ドキュメント、文面、ページ構成、設定変更のような作業にも使います。

この Skill の主責務は「既知の前提と制約をもとに成果物を作る」ことです。前提の発見は `research`、実施後の確認は `verification` に寄せます。呼び出し側は、できるだけ成果物の種類、対象読者、制約を明示します。

## クイックスタート

- 最初に `../../../AGENTS.md` を読む。
- stable なルール確認のために `../../../RULES.md` を読む。
- current truth の確認のために `../../../WORKING-CONTEXT.md` を読む。
- 何を作るのか、制約は何か、どこまで仕上げるかを短く整理する。
- `references/creation-workflow.md` を読んで、変更方針と締め方を揃える。

## 基本フロー

1. 対象の成果物、制約、完成条件を整理する。
2. 既存の構成や文脈を読み、変更方針を決める。
3. 必要な変更を作る。
4. 変更内容と残る前提を整理して次の確認につなぐ。
