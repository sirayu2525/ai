# Creation Workflow

## 着手前

- 何を作るのか、誰向けか、どこまで仕上げるかを確認する。
- 既存の構成、トーン、デザイン、実装方針があるなら先に読む。
- 目先の差し替えではなく、全体の一貫性を崩さない方針を決める。

## 作成中

- 部分修正よりも、成果物全体として自然かどうかを優先する。
- 必要なら責務分離、構成整理、見せ方の再設計まで含める。
- 変更理由が説明できる形で作る。

## 境界

- `creation` は成果物を作る責務を持つ。
- 何を作るべきか、どの前提を採用すべきかが不明なら `research` に戻る。
- 作ったものが妥当か、動くか、壊していないかの確認は `verification` に渡す。

## `research` への横移動

- 前提不足、仕様の曖昧さ、既存挙動の不明点、より良い実装方法やベストプラクティスの確認が必要になったら、`research` に戻ってよい。
- その場合も主目的は作成なので、調査は作るために必要な範囲に留める。
- `research` から戻るときは、追加で分かった前提、採用する方針、採用しなかった選択肢を整理してから作成を続ける。

## 締め方

- 何を作ったかをユーザー影響ベースで短くまとめる。
- 残る前提や未確認点があれば明示する。
- 次に必要な確認があれば添える。
