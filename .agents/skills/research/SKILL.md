# Research

## 概要

対象を理解し、必要な証拠を集めて、結論と未確定事項を整理するための基本 Skill です。コード変更の前段にも、サイト調査や比較検討にも使います。本題から逸れない範囲で、用語、背景、関連する考え方などの周辺知識も補足として拾います。

この Skill の主責務は「何を知る必要があるかを整理し、必要な証拠を集めて、判断材料を持ち帰る」ことです。ブラウザ上の一次証拠取得や UI 操作確認そのものが主題なら、`browser-debugging` や `web-flow-inspection` に委譲します。

## クイックスタート

- repo-wide docs は `../../../AGENTS.md` の方針に従って確認する。
- 調べたい対象、知りたいこと、返したい成果物を先に短く整理する。
- `references/research-workflow.md` を読んで、証拠の集め方、補足知識の扱い方、まとめ方を揃える。

## 基本フロー

1. 何を知りたいのか、どこまで確定させる必要があるのかを決める。
2. 関連するファイル、画面、ログ、外部資料から証拠を集める。
3. 理解を深める価値がある場合は、周辺知識や背景も補足として拾う。
4. 観測事実、推測、補足知識を分ける。
5. 結論、未確定事項、次の確認点を整理する。
