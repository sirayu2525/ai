# 進め方の例

## Playwright MCP で体験を確認する

- 「この URL を開いて、初見のユーザーとして違和感がある点を挙げて」
- 「トップページから会員登録完了まで進んで、詰まる導線を教えて」
- 「モバイル幅を意識して見て、情報の優先順位や CTA の見え方を評価して」
- 「フォーム操作を試して、入力体験やエラー表示の UX を見て」

## Chrome DevTools MCP で根拠を取る

- 「この画面が重い原因を network と performance の観点から見て」
- 「console error や warning が UX に影響していないか見て」
- 「DOM と computed style を見て、表示崩れの根本原因を探して」
- 「API の遅延や失敗が UI にどう影響しているか確認して」

## 結果のまとめ方

- 「重要度順に改善案を出して」
- 「ユーザー影響が大きい順に並べて」
- 「短期で直せるものと根本対応が必要なものを分けて」

## 使い分け

- まずは Playwright MCP で実際の体験を見る。
- 次に Chrome DevTools MCP で根拠を取る。
- 原因調査は `browser-analysis` に渡す。
