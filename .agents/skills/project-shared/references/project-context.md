# プロジェクト前提

このリポジトリにチェックインされた Skill と reference を、AI 作業フローの共有された正本として扱う。

## 基本方針

- 提案や編集に入る前に `README.md` と依頼に関係するファイルを読む。
- 変更前に現在のリポジトリ状態を確認する。
- 既存パターンに縛られず、コードがより整理されるなら抽象化や責務分割、実装方針の刷新を許容する。
- 変更範囲は適切に絞りつつも、実装品質の基準は下げず、より良い設計と保守性を常に意識する。
- 必要であればゼロベースで構造を見直し、より筋の良い設計へ再構成してよい。
- ツール固有のラッパーは薄く保ち、運用ルールが変わるときは shared reference を先に更新する。

## この Scaffold の育て方

- 長く使う cross-tool のルールはこの reference 群に置く。
- `SKILL.md` は導線とワークフロー説明に集中させる。
- 同梱 script は、繰り返し作業を減らすか、ミスを減らせるときだけ追加する。
- ツール要件がない限り、同じルールを `CLAUDE.md`、`SKILL.md`、他のラッパーに重複して書かない。
