# レポート形式

## 基本構成

1. 対象
2. 再現条件
3. 観測事実
4. 原因
5. 修正方針
6. 未確定事項 / 追加確認

## 書き方

- 観測事実では、実際に確認した console / network / DOM / screenshot / performance の内容を書く。
- 原因は、確定しているものと有力な仮説を分ける。
- 修正方針は、場当たり対応ではなく、再発を防ぐ方向で書く。
- 追加確認が必要なら、何を見れば確定できるかまで書く。

## 避けること

- 証拠なしに断定すること。
- 現象と原因を混同すること。
- 「問題なさそう」のような曖昧な締め方をすること。
- ただのログ列挙で終わること。
