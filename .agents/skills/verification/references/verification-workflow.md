# Verification Workflow

## 着手前

- 何を検証するのかを明確にする。
- 必要な信頼度に応じて、どこまで確認するかを決める。
- 自動確認、手動確認、目視確認のうち適切なものを選ぶ。

## 進め方

- まずは狭く有効な確認から始める。
- 失敗時は、原因切り分けに必要な追加確認を行う。
- すべてを網羅するより、依頼に対して妥当な確認を優先する。

## 境界

- `verification` は確認を実行して、観測結果を持ち帰る。
- その結果をどう評価し、どの問題を重く見るかは `review` に委ねる。
- 前提が不足して確認内容自体を決められない場合は `research` に戻る。

## まとめ方

- 実施した確認を明示する。
- 実施できなかった確認も明示する。
- 断定に足りない点があれば、残るリスクとして添える。
