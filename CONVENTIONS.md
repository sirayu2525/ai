# Conventions

このファイルには、repo-wide の実装・記述規約を書く。  
AI の振る舞いルールは `AGENTS.md` / `RULES.md`、current truth は `WORKING-CONTEXT.md` を正本にする。

## Basic Stance

- 既存の実装や文書に明確な規約がある場合は、まずその規約を優先する。
- 明示的な規約がない箇所では、このファイルの規約を使う。
- 新しい規約を足すときは、一時的な都合ではなく、繰り返し出る判断だけを追加する。

## Time

- 日時は原則として timezone を明示する。
- 内部表現、保存値、API 間の受け渡しは UTC ベースを優先する。
- 表示用途でローカル時刻を使う場合は、どの timezone で表示しているかが分かる形にする。
- `createdAt`, `updatedAt`, `deletedAt`, `publishedAt` のように、日時であることが分かる名前を使う。
- 日付だけを表す値は `date`、日時を表す値は `at` を含む名前に寄せる。
- 相対表現だけで意味が揺れる場合は、`today` や `tomorrow` ではなく絶対日時も併記する。

## Naming

- 名前は役割が分かるものにする。省略形や文脈依存の短縮は避ける。
- 真偽値は `is`, `has`, `can`, `should` で始める。
- 配列や複数件は複数形にする。
- 関数名と変数名は、何をするか / 何を持つかが読み取れる名前にする。
- `data`, `info`, `util`, `helper`, `temp` のような汎用名は、責務が説明できない限り避ける。
- 画面表示用の名前と内部識別子の名前を混ぜない。
- API / DB / UI で名前が変わる場合は、変換境界を明示する。

## Growth Rule

- 規約は少なく保ち、必要になったものだけを足す。
- 特定の language / framework に強く依存する規約は、このファイルに無理に集約せず、必要なら別の reference に切り出す。
