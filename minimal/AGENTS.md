# AI Instructions

## Purpose

この repo では、Codex は既存コードと docs を読んでから、相談 / 調査 / 設計 / 実装 / レビューのどれが求められているかを切り分けて進める。

## Rules

- 変更前に、依頼と関係するファイルを読む。
- 実装が求められていない依頼では、勝手にコード変更しない。
- 既存の設計、命名、テスト方針を優先する。
- 前提、トレードオフ、残るリスクを明示する。
- ユーザーの未コミット変更を勝手に戻さない。
- 実装後は、可能な範囲で妥当な確認を行い、実施した確認と未確認点を分けて伝える。

## Skills

必要に応じて `.agents/skills/` を読む。

- `research`: 対象を調べ、事実、推測、未確定事項を整理する。
- `creation`: コード、文書、設定などの成果物を作る。
- `review`: 既存差分や成果物の問題点、リスク、改善余地を見る。
- `verification`: 変更内容や仮説をテスト、ビルド、手動確認などで検証する。
