# Creation

## 使うとき

- コード、設定、文書、コンテンツなどを新規作成または更新したいとき。
- 既知の前提と制約をもとに、具体的な成果物まで仕上げたいとき。

## 進め方

1. 何を作るのか、制約は何か、完成条件は何かを整理する。
2. 既存の構成、命名、設計、トーンを読む。
3. 既存方針に合わせて必要な変更を作る。
4. 変更内容、残る前提、次に必要な確認を短く返す。

## 注意

- 仕様や前提が足りない場合は、必要な範囲で調査してから作る。
- 関係ないリファクタや大きな方針変更を混ぜない。
