# Verification

## 使うとき

- 実装後の挙動や、調査で立てた仮説を確認したいとき。
- テスト、ビルド、lint、手動確認、目視確認などを選んで実行したいとき。

## 進め方

1. 何を確認したいのか、どの程度の信頼度が必要かを整理する。
2. 最も狭く有効な確認から始める。
3. 失敗した場合は、原因切り分けに必要な追加確認を行う。
4. 実施した確認、結果、未実施の確認、残るリスクを分けて返す。

## 注意

- 確認結果と評価を混ぜない。
- 実行できなかった確認は隠さず伝える。
